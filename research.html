Explore all the other input tags

The input type="reset" defines a reset button that will reset all form values to their default values

The input type="color" is used for input fields that should contain a color. Depending on browser support, a color picker can show up in the input field.

The input type="datetime-local" specifies a date and time input field, with no time zone.Depending on browser support, a date picker can show up in the input field.

The input type="hidden" defines a hidden input field (not visible to a user).A hidden field lets web developers include data that cannot be seen or modified by users when a form is submitted.A hidden field often stores what database record that needs to be updated when the form is submitted.

The input type="range" defines a control for entering a number whose exact value is not important (like a slider control). Default range is 0 to 100. However, you can set restrictions on what numbers are accepted with the min, max, and step attributes.

The input type="search" is used for search fields (a search field behaves like a regular text field).

The input type="time" allows the user to select a time (no time zone).Depending on browser support, a time picker can show up in the input field.

The input type="url" is used for input fields that should contain a URL address.Depending on browser support, the url field can be automatically validated when submitted.Some smartphones recognize the url type, and adds ".com" to the keyboard to match url input.

The input type="week" allows the user to select a week and year.Depending on browser support, a date picker can show up in the input field.

About get and post values in method attribute in forms

The GET method sends form data appended to the URL as query parameters, making the data visible in the URL itself. This method is useful when the action does not alter the server's state, such as in search forms or when the results need to be bookmarked. However, it is important to note that GET is limited by URL length restrictions, which means itâ€™s only suitable for small amounts of data. For example, in a search form where a user enters a query, submitting the form would result in a URL like /search?query=example, with the data clearly visible.

On the other hand, the POST method sends form data in the body of the HTTP request rather than in the URL. This approach is more secure for transmitting sensitive information, such as passwords or personal details, and can also handle larger amounts of data without the constraints of URL length. POST is typically used for forms that modify the server's data, such as user registration, login forms, or any action that updates the database. When using POST, the form data is not exposed in the URL, offering better protection for sensitive information.

why we use meta name="view"?

The tag helps ensure that a website is displayed correctly on all devices, especially mobile devices. It allows you to set the width and scaling of the viewport so that it's the right size for the device's screen. Without this tag, the content on a mobile device might be too small or squeezed to fit, making it difficult to read.

charset,property,http attributes in meta tags

charset: Defines the character encoding (e.g., UTF-8).
name and content: Provide metadata for SEO, authoring, and viewport settings.
property and content: Used for Open Graph metadata to control how content appears on social media.
http-equiv and content: Mimic HTTP headers to control document behavior like caching and compatibility.
